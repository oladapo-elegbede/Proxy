# PROXY
### Institutional Navigation for Neurodivergent Learners

**Live Demo:** https://proxy-kappa-pearl-66.vercel.app  
**Repository:** https://github.com/oladapo-elegbede/Proxy

---

## The Problem

The systems built to support neurodivergent learners require exactly the executive-function skills those learners struggle with most.

To receive accommodations for executive function impairment, a student must:
- Know which office to contact
- Know what documentation to gather
- Know how to describe their needs in institutional language
- Navigate a multi-step bureaucratic process
- Advocate for themselves under stigma and anxiety

These are precisely the capabilities most impaired in the students who need the system most.

**Fewer than 40% of neurodivergent students successfully access accommodations. The barrier is not awareness — it is the advocacy burden.**

PROXY removes that burden entirely.

---

## What PROXY Does

PROXY is an AI-powered institutional navigation system.

A student describes what is hard — in their own words, no forms, no clinical language required. PROXY identifies their functional barriers, maps them to the support they are legally entitled to, and generates a complete, step-by-step pathway through their institution's accommodation process.

At every step, PROXY tells the student exactly what to do and exactly what to say.

    Why AI is essential here — not decorative:

Every AI call in PROXY performs a task that rules-based software cannot:

Intake Pipeline — takes a student's messy, emotional, plain-language description and maps it to specific functional barriers from a validated taxonomy. No keyword matching can do this reliably. The student may say "I just freeze" and the AI correctly identifies task initiation impairment.

Pathway Pipeline — takes barrier IDs and institution process steps from the Knowledge Graph and sequences them into a coherent, student-facing pathway with warm language. The AI does not invent steps — it sequences and narrates what the graph already knows.

Conversation Architect — generates a personalised preparation script calibrated to the student's specific situation, the specific institutional conversation they are about to have, and language that feels like advice from a supportive friend who knows the system.

        Neurodivergent Design Principles
Every interaction was designed against these non-negotiable principles:

Principle	Implementation
Zero prerequisite knowledge	Student never needs to know institutional terminology
Zero forms before trust	No intake form — open conversation first
One next step at a time	Pathway shows one active step, future steps visible but inactive
Shame has no home here	No clinical language, no judgment, no "why didn't you register earlier"
Student owns all data	Nothing sent anywhere without explicit student approval
Preparation creates confidence	Conversation scripts built before every institutional interaction
Accessibility
PROXY was built accessibility-first, not accessibility-retrofitted.

Semantic structure:

Correct landmark regions (<main>, <nav>, <section>, <footer>)
Logical heading hierarchy (h1 → h2 → h3) throughout
Ordered and unordered lists used semantically
Definition lists (<dl>, <dt>, <dd>) for conversation script sections
Screen reader support:

aria-live="polite" on all streaming and dynamic content
aria-live="polite" on loading states
role="alert" on all error messages
role="status" on success states and completion banners
role="progressbar" with aria-valuenow, aria-valuemin, aria-valuemax
All decorative emoji have aria-hidden="true"
Step completion and active status announced via sr-only text
Keyboard navigation:

All interactive elements reachable by Tab
Submit on Ctrl+Enter in textarea
Skip-to-content link as first focusable element
focus:ring-2 visible focus indicators on all interactive elements
Forms:

Visible <label> elements with htmlFor
aria-invalid on invalid inputs
aria-required on required fields
aria-describedby linking inputs to prompts and errors
Character count announced via aria-live when near limit
Responsive:

Tested at 320px, 375px, 768px, 1024px, 1280px+
Mobile-first layout — stacks vertically, side by side on larger screens
iOS safe area padding on fixed bottom bar
UX Resilience
Every async operation has all four states:

Operation	Loading	Success	Failure	Retry
Intake AI call	Streaming animation	Response displayed	Student-friendly error	Re-submit
Pathway generation	Animated dots + explanation	Redirect to pathway	Error + retry button	One click retry
Conversation script	Animated dots	Script displayed	Error message	Button reappears
Global crashes	N/A	N/A	Error boundary page	Reset or go home
No blank screens. No dead ends. Every error message is written for a student, not a developer.
